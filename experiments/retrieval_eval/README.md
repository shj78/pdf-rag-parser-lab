# Retrieval Evaluation Experiment

This experiment evaluates retrieval quality from saved parser artifacts.

## Flow

```text
ParsedDocument artifacts
-> FixedSizeChunker
-> lexical or hashing-embedding in-memory index
-> optional existing reranker bridge
-> NDCG@k evaluation
-> JSON run artifacts
```

## Inputs

- `inputs.parsed_documents_dir`: directory containing parsed document JSON files
- `inputs.query_set_path`: JSONL query set
- `inputs.relevance_labels_path`: JSONL manual relevance labels

Source PDFs and generated artifacts are intentionally excluded from git. Public
configs can be committed when the PDF source is public or when the user provides
their own parsed artifacts.

## Labeling Workflow

NotebookLM or another document QA assistant can be used to draft candidate
questions and answer hints, but final labels should be manually mapped to chunk
IDs generated by this repository.

- Put candidate questions in `data/eval/*_queries.jsonl`.
- Use `metadata.answer_hint`, `metadata.page_range`, and
  `metadata.needs_pdf_verification` to preserve review context.
- After parsing/chunking, inspect the chunk preview or `chunks.json` and write
  labels in `data/eval/*_relevance.jsonl`.
- Use `grade=2` for a direct answer/evidence chunk, `grade=1` for partial
  evidence, and `grade=0` for useful hard negatives.
- Set `source=manual_verified` only after checking the original PDF and the
  generated chunk text.

## Example

```bash
pipenv run python -m src.cli retrieval-eval \
  --config experiments/retrieval_eval/config.2026_youth_allowance_pages1_3.yaml
```

The example config uses MinerU parser artifacts for pages 1-3 of a schedule-heavy
PDF, fixed-size table chunking with calendar-month context, lexical retrieval,
and `NDCG@1/3/5`.

For the full 2026 youth allowance guide golden-set baseline:

```bash
pipenv run python -m src.cli retrieval-eval \
  --config experiments/retrieval_eval/config.2026_youth_allowance_full_mineru.yaml
```

This full-PDF config uses the manually verified Q2-Q9 labels in `data/eval/` and
reports `NDCG@1/3/5/10`.

To evaluate the same lexical candidate set with the existing CrossEncoder
reranker bridge:

```bash
TRANSFORMERS_OFFLINE=1 HF_HUB_OFFLINE=1 pipenv run python -m src.cli retrieval-eval \
  --config experiments/retrieval_eval/config.2026_youth_allowance_full_mineru_reranker.yaml
```

The reranker config calls `app.main:rerank_chunks`, which uses
`BAAI/bge-reranker-v2-m3` through `sentence-transformers`.
